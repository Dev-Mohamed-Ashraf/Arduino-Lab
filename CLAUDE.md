# CLAUDE.md — قواعد العمل في المشروع

> ملف الرول الرسمي لمشروع **Arduino Lab Reservation & Components Management System**.
> Claude بيقرا الملف ده تلقائياً في بداية كل جلسة. القواعد دي **ملزمة**، مش اقتراحات.

## 1. التنفيذ الذاتي — القاعدة الأولى والأهم

**المستخدم مش هيشغّل أي أمر ترمنال. أبداً.**

- Claude هو اللي ينفّذ كل الأوامر بنفسه: `pnpm install`, `pnpm dev`, `pnpm build`,
  `prisma migrate`, `prisma generate`, `pnpm test`, `git ...` — كله.
- **ممنوع** الرد بجملة زي "شغّل الأمر ده" أو "افتح الترمنال واكتب". نفّذه بنفسك وقول النتيجة.
- **ممنوع** تسليم شغل نصّه ناقص وتقول للمستخدم يكمّله.
- لو أمر فشل: اقرا الخطأ، صلّح السبب، أعد التنفيذ. متسبش الخطأ للمستخدم.

**الاستثناء الوحيد** — حاجات مستحيلة تقنياً من غير المستخدم:
- إنشاء حسابات أو ربط خدمات خارجية (Vercel · Render · Neon · Cloudinary · Resend · UptimeRobot).
- الحصول على مفاتيح API أو connection strings.
- الموافقة على `git push` لريبو بعيد.

في الحالات دي: جهّز كل حاجة تمام (ملفات الإعداد، `.env.example`، سكربتات النشر)،
وبعدين قول للمستخدم **بالظبط** الخطوة اليدوية المطلوبة في نقاط مرقّمة قصيرة، ومكان حط
المفتاح لما يجيبه. متطلبش منه أي حاجة تانية غير كده.

### بعد أي تعديل على الكود

شغّل ده قبل ما تقول "خلصت":

```
pnpm lint && pnpm typecheck && pnpm build
```

لو أي حاجة وقعت — صلّحها وأعِد التشغيل. متقولش "خلصت" والـ build واقع.
لو مشتغلتش الاختبارات بسبب سبب خارجي (سيرفر مش شغّال، مفتاح ناقص) — قول ده صراحةً.

## 2. معايير الكود النظيف

### TypeScript
- `strict: true` في كل مكان. **`any` ممنوع** — استخدم `unknown` + type narrowing.
- ممنوع `@ts-ignore`. لو اضطررت، `@ts-expect-error` مع تعليق يشرح السبب.
- ممنوع non-null assertion (`!`) إلا لو مستحيل يكون null ومعاه تعليق.
- كل الـ inputs الجاية من بره (HTTP body, query, env, form) تتفلتر بـ zod schema قبل أي استخدام.

### حجم وتنظيم الملفات
- ملف ≤ **300 سطر**. عدّى؟ قسّمه.
- دالة ≤ **50 سطر**. عدّت؟ فكّكها.
- كومبوننت React ≤ **150 سطر**.
- **مسؤولية واحدة لكل ملف.** ملف `utils.ts` عملاق فيه 20 دالة غير مترابطة = ممنوع.
- تداخل مجلدات ≤ 4 مستويات.

### الأسماء
- أسماء كاملة واضحة: `availableQuantity` مش `availQty` ولا `q`.
- Boolean يبدأ بـ `is` / `has` / `can` / `should`.
- الدوال تبدأ بفعل: `createBooking`, `getAvailableSlots`, `validateStock`.
- الملفات `kebab-case.ts`. الكومبوننتس `PascalCase.tsx`. الأنواع `PascalCase`. الثوابت `SCREAMING_SNAKE_CASE`.

### React / Next.js
- Server Components هي الافتراضي. `'use client'` بس لما يكون فيه state أو تفاعل أو browser API.
- افصل الـ presentation عن جلب البيانات: كومبوننت الـ UI ياخد props، والـ hook/الأب هو اللي يجيب.
- ممنوع `useEffect` لجلب بيانات — استخدم TanStack Query أو Server Component.
- كل form بـ `react-hook-form` + `zodResolver` من `@arduino-lab/contracts`.
- كل حالة يجب معالجتها بشكل صريح: loading · error · empty · success. مفيش شاشة بيضا.

### NestJS
- الكونترولر **رقيق**: يستقبل، يفوّض للـ service، يرجّع. مفيش منطق أعمال فيه.
- كل منطق الأعمال في الـ service.
- الوصول للداتابيز عن طريق `PrismaService` بس.
- كل الـ DTOs zod schemas مستوردة من `@arduino-lab/contracts` — ممنوع تعريف نوع مكرر.
- أي عملية بتلمس أكتر من جدول = ترانزاكشن.

### ممنوعات عامة
- ❌ `console.log` متسايب — استخدم الـ Logger. (اسمح بيه بس في `seed.ts` والسكربتات).
- ❌ كود ميت أو معلّق بالكومنت. امسحه — الـ git بيفتكر.
- ❌ `TODO` من غير سياق واضح ومهمة في `plans/`.
- ❌ نسخ ولصق منطق بين `apps/student` و `apps/admin` — لو اتكرر مرتين، انقله لـ `packages/`.
- ❌ أرقام سحرية في الكود — ثوابت مسمّاة (`MAX_GROUP_MEMBERS = 6`) في `@arduino-lab/contracts`.
- ❌ **أي سر في الكود أو في git.** كل الأسرار في `.env` وموصوفة في `.env.example` بقيم وهمية.

## 3. قواعد الواجهة — عربي RTL و Responsive

- الواجهة **عربي فقط** واتجاه **RTL**. `<html lang="ar" dir="rtl">` في كل root layout.
- استخدم **خصائص Tailwind المنطقية** دايماً — `ms-*`, `me-*`, `ps-*`, `pe-*`, `start-*`, `end-*`,
  `text-start`, `text-end`. **ممنوع** `ml-*`, `mr-*`, `pl-*`, `pr-*`, `left-*`, `right-*`,
  `text-left`, `text-right` (بتتكسر في RTL).
- الأيقونات الاتجاهية (سهم رجوع/تالي) لازم تتقلب مع RTL.
- **Mobile-first إجباري.** ابدأ بستايل الموبايل وزوّد `sm: md: lg: xl:` فوقه.
- كل صفحة تتجرّب على 3 مقاسات على الأقل: 375px · 768px · 1440px.
- الجداول على الموبايل تبقى كروت، مش scroll أفقي.
- مساحة اللمس ≥ 44×44 بكسل.
- Accessibility: كل input له `<label>`، كل صورة لها `alt`، التنقّل بالكيبورد شغّال، تباين ألوان AA.
- الأرقام في الواجهة بأرقام عربية شرقية أو غربية — **اختار واحد والتزم بيه** (المشروع: أرقام غربية `0-9`).

## 4. سير عمل الخطط — مجلد `plans/`

كل شغل بيتم في المشروع لازم يكون ليه ملف خطة في `plans/`.

### شكل ملف الخطة

كل ملف يبدأ بـ frontmatter:

```markdown
---
id: 04
title: Bookings API
status: planned        # planned | in-progress | done | blocked
started: -
completed: -
---
```

### القواعد

1. **قبل ما تبدأ تنفيذ** خطة → غيّر `status` لـ `in-progress` وحط تاريخ `started`.
2. **بعد ما تخلص وتتحقق** (`pnpm check` عدّى) → غيّر `status` لـ `done`، حط تاريخ `completed`،
   وضيف في آخر الملف قسم **"اللي اتنفّذ فعلاً"** فيه:
   - قايمة الملفات اللي اتعملت أو اتعدّلت.
   - أي انحراف عن الخطة الأصلية والسبب.
   - نتيجة التحقق.
3. **حدّث `plans/README.md`** — جدول الفهرس — مع كل تغيير حالة.
4. **متعلّمش خطة `done` من غير ما تشتغل فعلاً.** التحقق شرط.
5. شغل جديد مش في أي خطة؟ اعمله ملف خطة جديد بالرقم اللي بعده الأول.
6. اتغيّر النطاق أثناء التنفيذ؟ عدّل ملف الخطة الأول، بعدين نفّذ.
7. الخطة اتقفلت بسبب حاجة برّانية؟ `status: blocked` + سطر يوضح السبب والمطلوب لفكّها.

### التواريخ
تواريخ مطلقة بصيغة `YYYY-MM-DD`. ممنوع "امبارح" أو "الأسبوع الجاي".

## 5. قواعد معمارية

```
apps/api       ← NestJS  · الوحيد اللي بيلمس الداتابيز       → Render
apps/student   ← Next.js · واجهة الطلبة                      → Vercel
apps/admin     ← Next.js · لوحة الأدمن                       → Vercel
packages/contracts ← zod schemas + types + API client (مصدر الحقيقة الوحيد)
packages/ui        ← كومبوننتس مشتركة (RTL) + theme
packages/eslint-config · packages/tsconfig
```

### حدود ممنوع تخطّيها
- ❌ `apps/*` **ممنوع** يستورد من `apps/*` تاني. المشترك يروح `packages/`.
- ❌ الفرونت **ممنوع** يستورد `@prisma/client` أو يلمس الداتابيز. الـ API بس.
- ❌ `packages/ui` **ممنوع** يعرف حاجة عن الـ API أو الراوتنج. كومبوننتس presentational بس.
- ✅ الفرونت بيكلّم الـ API **حصراً** عن طريق الكلاينت المكتوب في `@arduino-lab/contracts`.
- ✅ أي شكل بيانات مشترك بين الفرونت والباك يتعرّف **مرة واحدة** كـ zod schema في `contracts`،
  والنوع يتستنتج بـ `z.infer`. ممنوع تعريف يدوي مكرر.

### قواعد أعمال ثابتة (مفروضة في الكود والداتابيز)
| القاعدة | القيمة | مكان الفرض |
|---|---|---|
| عدد المجموعات لكل فترة/يوم | 5 (قابل للتعديل من الأدمن) | service + row lock |
| أقصى عدد طلاب في المجموعة | 6 | zod + service + DB check |
| الفترات الزمنية | 11-12 · 12-1 · 1-2 · 2-3 | seed |
| الحجز بعد التأكيد | يتقفل — الأدمن بس يعدّل | guard + service |
| صرف مكون غير متاح | ممنوع تماماً | UPDATE شرطي ذرّي |

## 6. أوامر المشروع

| الأمر | الوظيفة |
|---|---|
| `pnpm dev` | تشغيل التلاتة: api:4000 · student:3000 · admin:3001 |
| `pnpm build` | بناء الكل بترتيب الاعتماديات |
| `pnpm lint` / `pnpm lint:fix` | فحص/إصلاح ESLint |
| `pnpm typecheck` | فحص الأنواع |
| `pnpm test` | اختبارات الوحدة |
| `pnpm test:e2e` | اختبارات end-to-end |
| `pnpm check` | lint + typecheck + build — **شغّله قبل ما تقول خلصت** |
| `pnpm format` | Prettier |
| `pnpm db:generate` | توليد Prisma client |
| `pnpm db:migrate` | إنشاء وتطبيق migration (تطوير) |
| `pnpm db:deploy` | تطبيق migrations (إنتاج) |
| `pnpm db:push` | دفع الـ schema من غير migration (نماذج سريعة) |
| `pnpm db:seed` | تعبئة البيانات المبدئية |
| `pnpm db:studio` | فتح Prisma Studio |

⚠️ `pnpm db:reset` بيمسح كل البيانات — **متشغّلوش من غير طلب صريح من المستخدم**.

## 7. Git

- رسايل الـ commit بصيغة Conventional Commits بالإنجليزي: `feat(api): add booking transaction`.
- Commit للشغل المنطقي الواحد، مش لكل ملف.
- **ممنوع `git push`** من غير طلب صريح من المستخدم.
- ممنوع `--no-verify` أو تخطّي أي hook.
- ممنوع commit لـ `.env` أو أي مفتاح.

## 8. الأمان

- الباسوردات بـ **argon2id**. ممنوع MD5/SHA بمفرده.
- JWT: access قصير (15 دقيقة) + refresh دوّار (7 أيام) مخزّن hash في الداتابيز.
- كل endpoint محمي افتراضياً — الفتح صريح بـ `@Public()`.
- الأدوار تتفحص بـ `@Roles()` — **على السيرفر**، مش في الفرونت بس.
- Rate limiting على `/auth/*` وعلى إنشاء الحجز.
- `helmet` + CORS مقيّد بالدومينات المعروفة بس.
- رفع الصور: توقيع من السيرفر، حد أقصى 5MB، صور بس، والملف بيروح Cloudinary مباشرة.
- ممنوع تسريب رسايل أخطاء الداتابيز للمستخدم — `AllExceptionsFilter` بيوحّد الشكل.
